## 1. ゲームバランス

### 1.1 リソース二層制

本作のリソースは性質の異なる2種類で構成し、「持続的な行動力」と「決定的な投資力」を分離する。

| リソース | 略称 | 型 | 入手 | 用途 | 性格 |
| --- | --- | --- | --- | --- | --- |
| Compute Power | **CP** | フロー型（毎秒回復・上限あり） | 時間経過で自動回復 | AIの **行動コスト**（攻撃・スキャン・修復等） | 「今この瞬間に何手打てるか」 |
| Data Points | **DP** | ストック型（貯蓄） | 攻撃成功・防御成功・解析進行 | ノードの **設置・増設コスト**、維持費 | 「盤面にどれだけ投資できるか」 |

- **CP** は使い切っても回復する。だが上限があるため「同時に撃てる手数」は常に有限。AIが欲張れば手詰まりになる。
- **DP** は貯め込めるが、稼ぐには戦闘で能動的にリスクを取る必要がある。守るだけでは増えにくい。

この二層構造により、「守って待つ」だけの消極戦術が成立せず、攻めなければDPが枯れ、攻めすぎればCPが枯れる、という緊張を常時生む。

### 1.2 初期数値表（チューニングのベースライン）

> 以下はラウンド開始時の標準値。バランス調整の基準点として用いる。

| パラメータ | 値 | 補足 |
| --- | --- | --- |
| 初期DP | **1000** | ビルドフェーズの初期構築原資 |
| CP回復 | **+10 / 秒**（上限 **100**） | 0秒時点の上限。後述の漸増あり |
| 抽出速度（基礎） | **0.5% / 秒** | コアに到達した攻撃ノードが1つ稼働している場合 |
| 抽出のコア距離減衰 | **1ホップごとに ×0.85** | 経路が深い（多層）ほど抽出が遅くなる（後述 1.5②） |
| 攻撃成功報酬 | 撃破ノードの維持DPの **30%** を獲得 | 例: 維持DP=200のノード撃破 → +60DP |
| 防御成功報酬 | **+5 DP** ＋ **CP +3**（即時） | 攻撃を撃退/無力化したノードに付与 |
| 解析進行報酬 | スキャン1段階ごと **+8 DP** | 敵トポロジーの可視化（霧の除去）に連動 |
| CP上限の漸増 | **90秒以降、+2/秒ずつ上限が上昇**（最大160） | 終盤を加速させ膠着を防ぐ |
| ラウンド制限時間 | **180秒** | 時間切れ時は抽出%が高い側の勝ち。同%なら残コアHP比較 |

#### 抽出速度の計算例

機密データへ到達した攻撃ノードが、コアから3ホップ離れた経路で稼働している場合:

```
抽出速度 = 0.5% × (0.85 ^ 3)
        = 0.5% × 0.614
        ≈ 0.307% / 秒
→ 単純計算で 100% まで約 326 秒（ただし複数経路・複数ノードで合算され短縮される）
```

複数の侵入経路を同時に通せば抽出が合算されるため、攻撃側は「単発の深い貫通」か「複数の浅い同時侵入」かの設計判断を迫られる。

### 1.3 ノードの三すくみ設計（コスト ⇔ 役割 ⇔ カウンター）

各ノードは「役割」と「弱点（カウンター）」を持ち、ジャンケン的な相性関係を形成する。トポロジー設計とは、相手の構成を読んでこの相性を盤面上で噛み合わせる作業である。

#### 主要カウンター関係（三すくみの核）

```
   DDoS ──（無力化される）──▶ シンクホール
     ▲                            │
     │                       （吸収して逆に）
（押し切る）                        │
     │                            ▼
  ゼロデイ ◀─（貫通される）── ファイアウォール
     ▲                            ▲
     │                            │
（逆探知で潰す）            （守りを固める）
     │                            │
  ハニーポット ◀──（誘い込む）── スキャナー
```

- **DDoS → シンクホール** に無力化される（過負荷を吸い込まれ不発）
- **スキャナー → ハニーポット** に逆探知される（偽ノードに食いつき攻撃元を露呈）
- **ファイアウォール → ゼロデイ** で貫通される（既知防御は未知の脆弱性に弱い）

#### ノードコスト表

| ノード | 種別 | 設置DP | 維持DP/秒 | 稼働CP/秒 | 役割 | 弱点（カウンター） |
| --- | --- | --- | --- | --- | --- | --- |
| **コア (機密DB)** | 中枢 | — | — | — | 防衛対象。陥落=抽出対象 | （全て） |
| **ルーター** | 接続 | 80 | 1 | 0 | ノード間の接続ハブ。経路を分岐 | 単体では脆い |
| **ファイアウォール (FW)** | 防御 | 150 | 3 | 1 | 通過攻撃を減衰・遮断。チョークポイント化 | ゼロデイで貫通 |
| **シンクホール** | 防御 | 200 | 4 | 2 | DDoSトラフィックを吸収・無力化 | スキャナーで位置露呈 |
| **ハニーポット** | 防御/情報 | 120 | 2 | 1 | 偽の機密。スキャナーを誘引し逆探知 | DDoSで力押し突破 |
| **スキャナー** | 情報/攻撃 | 130 | 2 | 4（稼働中） | 敵トポロジーの霧を晴らす | ハニーポットに逆探知される |
| **DDoSキャノン** | 攻撃 | 180 | 3 | 6（発射中） | ノードHPを面で削る | シンクホールに無力化 |
| **エクスプロイトボット** | 攻撃 | 220 | 4 | 5（攻撃中） | 単体ノードを貫通・抽出を担う | FW/シンクホールで足止め |
| **リペアデーモン** | 支援 | 110 | 2 | 3（修復中） | 味方ノードHPを回復 | 攻撃集中で処理が追いつかない |
| **デコイ (囮)** | 攪乱 | 60 | 1 | 1（展開時） | 敵AIの攻撃対象優先度を誤誘導 | 看破されると無駄撃ち |

> コストは「設置DP（初期投資）」「維持DP/秒（盤面に置き続ける固定費）」「稼働CP/秒（行動した瞬間の変動費）」の3軸。強力なノードほど三重に高い。

### 1.4 トポロジー設計が効く理由

- ノードは **接続された経路上でしか機能しない**。FWを置いても、攻撃が通らない経路に置けば無駄。
- コアへの経路は複数作れるが、経路が増えるほど維持DPがかさみ、守る面も広がる。
- 「1本の深い経路（多層防御だが抽出されると一気）」vs「複数の浅い経路（分散だが各個撃破に弱い）」のトレードオフがトポロジー設計の核心。

### 1.5 「白熱」を生む3つの仕組み

ゲームが一方的にならず、最後まで競るための設計上の安全装置。

1. **CP上限による同時行動制限**
   CPに上限があるため、AIは「全ノードを同時にフル稼働」できない。攻めれば守りのCPが、守れば攻めのCPが足りなくなる。常に「今この瞬間、何を優先するか」の取捨選択が発生し、リソース無双が起きない。

2. **コア距離ホップ減衰で多層設計に意味**
   抽出はコアから遠いほど（×0.85^ホップ）遅くなる。攻撃側は「速いが浅い経路（守りやすい）」と「深く潜って速度を稼ぐ（時間がかかる）」のジレンマを抱える。防御側は層を増やすほど抽出を遅延できるため、トポロジーの深さそのものが防御リソースになる。一撃必殺が成立しない。

3. **攻撃/防御の相互リソース報酬で一方的展開を緩和**
   攻撃成功は攻撃側にDPを与えるが、防御成功は防御側にDP＋CPを与える。つまり攻め込まれている側も、撃退するたびに反撃の原資が貯まる。劣勢側に自然なカムバック原資が供給され、勝者総取りの雪だるま式崩壊を防ぐ。

## 2. AI作戦ポリシー（IF-THEN）

### 2.1 基本構造

戦闘フェーズで自軍AIを動かすのは、プレイヤーがビルドフェーズで組んだ **優先順位付きルールリスト** である。

- ルールは **上から順に評価** され、条件を満たした最初のルールの行動を実行する（優先順位 = リストの並び順）。
- 条件・行動はそれぞれ **カード化** されており、ドラッグ＆ドロップで組み立てる。
- **ルール枠には上限がある**（標準: 12枠）。攻撃・防御・情報の全方位を詰め込むことはできず、「何を捨てるか」を競う。

```
ルールリスト（上から評価）
┌────┬──────────────────────────────┬───────────────────────────┐
│ #  │ IF（条件カード）               │ THEN（行動カード）          │
├────┼──────────────────────────────┼───────────────────────────┤
│ 1  │ 自軍コアHP < 30%              │ 全エクスプロイトをコア防衛へ │
│ 2  │ 敵抽出% > 70%                 │ コア防衛全振り＋リペア集中   │
│ 3  │ ハニーポット反応あり          │ 反応座標へDDoS集中          │
│ …  │ …                            │ …                          │
│ 12 │ （デフォルト）                │ 最寄りの敵ノードへ通常攻撃   │
└────┴──────────────────────────────┴───────────────────────────┘
```

### 2.2 条件カテゴリ（IF）

| カテゴリ | 例となる条件カード |
| --- | --- |
| 自軍HP | `自軍コアHP < X%`, `任意ノードHP < X%` |
| 自軍CP | `自CP < X`, `自CP > X%（余剰）` |
| 自軍DP | `自DP > X`, `自DP < X` |
| 敵可視率 | `敵可視率 < X%`, `敵トポロジー未解明` |
| 抽出% | `自軍抽出 > X%`, `敵抽出 > X%` |
| 経過時間 | `経過 > X秒`, `残り時間 < X秒` |
| イベント | `ハニーポット反応`, `同一ノードN回遮断成功`, `ゼロデイ被弾`, `スペシャルゲージ満タン` |

### 2.3 行動カテゴリ（THEN）

| カテゴリ | 例となる行動カード |
| --- | --- |
| 攻撃対象選択 | `最浅の敵ノードを攻撃`, `敵コア直行`, `FWを優先撃破`, `攻撃元へ反撃` |
| 防御集中 | `コア周辺へFW増設`, `指定ノードへ防御リソース集中` |
| 修復 | `最もHPの低い味方を修復`, `コアのみ修復` |
| 撤退 | `劣勢経路を切り離す`, `前線ノードを後退` |
| 囮展開 | `デコイ展開`, `偽抽出ノードで誘引` |
| 温存 | `攻撃停止しCP回復`, `行動を1tick待機` |
| スペシャル | `トレースバック発動`, `ゼロデイ発動`, `キルスイッチ発動` |

### 2.4 ルール具体例（12例）

> 実際に組み込める即運用可能なルール。上ほど優先度が高い想定。

| # | IF（条件） | THEN（行動） | 意図 |
| --- | --- | --- | --- |
| 1 | 自軍コアHP < 30% | 経路を切り離してコア防衛に全振り | 崩壊回避を最優先 |
| 2 | 敵抽出 > 70% | コア防衛全振り＋全リペア稼働 | 抽出完了の阻止 |
| 3 | ハニーポット反応あり | 反応座標（攻撃元）へDDoS集中 | 逆探知から殲滅へ転換 |
| 4 | 自CP < 20% | 攻撃停止しCP回復を待つ | 手詰まり防止・温存 |
| 5 | 敵可視率 < 50% | スキャン優先（攻撃より索敵） | 情報不足での突撃を回避 |
| 6 | 同一ノードを3回遮断成功 | その地点にFW増設しチョークポイント化 | 成功パターンの固定化 |
| 7 | 自軍抽出 > 50% かつ 残り時間 < 60秒 | 全エクスプロイトを抽出経路に投入し押し切り | 時間切れ勝ちの確定狙い |
| 8 | ゼロデイ被弾（FW貫通された） | 貫通経路を切り離し別経路で迂回防御 | 既知脆弱性の被害局所化 |
| 9 | スペシャルゲージ満タン かつ 劣勢 | トレースバック発動 | 逆転札の適切な切り時 |
| 10 | 敵がシンクホール展開を検知 | DDoSを停止しエクスプロイトに切替 | 無力化される攻撃の中止 |
| 11 | 余剰CP > 40% かつ 敵可視率 > 70% | デコイ展開で敵攻撃を誤誘導 | 余力を攪乱に転用 |
| 12 | （デフォルト/上記いずれも不成立） | 最浅の敵ノードへ通常攻撃しDP獲得 | 基本行動・リソース稼ぎ |

### 2.5 ルール枠の戦略的意味

12枠という上限により、プレイヤーは以下のような「思想」の取捨選択を迫られる。

- **攻撃寄せ型**: 攻撃対象選択ルールを多く積み、防御は最低限（コア崩壊時のみ反応）。速攻で抽出を狙う。
- **防御寄せ型**: 防御・修復ルールを厚くし、相手のCP枯れを誘って時間切れ判定を狙う。
- **情報寄せ型**: スキャン・デコイを軸に相手を誤誘導し、相性で勝つ。

枠は有限なので「全部入り」はできず、相手の型を読んでメタを張る読み合いが生まれる。これがビルドフェーズの主戦場である。

## 3. 逆転要素（スペシャル）

### 3.1 設計方針

- すべてのスペシャルは **溜め（ゲージ）＋明確な発動条件** を持ち、運要素は一切排除する。
- 効果は **強力だが必ずリスク（デバフ・自軍ペナルティ）を伴う**。
- 逆転札の発動が成立したときの **最終勝率が概ね50%前後** に収まるよう、発動条件を意図的に重くする（=「撃てば勝ち」にしない）。

### 3.2 スペシャル一覧

| スペシャル | 溜め条件 | 発動条件 | 効果 | リスク／コスト |
| --- | --- | --- | --- | --- |
| **トレースバック** | 解析率の蓄積 | 解析率 **100%** 到達で **一度だけ** 発動可 | 敵の主要ノードを数秒（標準4秒）停止 ＋ 敵CPをドレイン（-40CP） | 発動後は解析率0%にリセット。再充填に時間 |
| **ゼロデイ・エクスプロイト** | 劣勢時に貯まる「窮地ゲージ」 | 窮地ゲージ **満タン** で発動 | 効果A: FWを完全貫通（数秒） / 効果B: 敵抽出ゲージを巻き戻し（-20%）のいずれか選択 | 使用後、自軍全ノードに被デバフ（防御-25%, 10秒間） |
| **キルスイッチ** | 常時待機（条件のみ） | 自軍コア陥落 **寸前**（コアHP < 10%）でのみ発動可 | 機密データを自己暗号化し、**敵の抽出%を0%に強制リセット** | 自軍も罰: 自軍の抽出%も0%に＋自軍CP上限を半減（残りラウンド永続）。完全な「相打ち・延命」札 |

### 3.3 窮地ゲージの設計（ゼロデイ用）

劣勢側にのみ逆転原資が貯まるよう、ゲージは「不利な状況」で増える。

```
窮地ゲージ増加 = base
  + (敵抽出% - 自抽出%) × 係数a        // 抽出で負けているほど増
  + (失ったノード数) × 係数b           // ノードを失うほど増
  - (自軍が優勢な時間) × 係数c         // 優勢だと減衰

→ 圧倒的劣勢でのみ満タンになる設計。優勢側は基本撃てない。
```

### 3.4 勝率50%設計の考え方

- スペシャルは「劣勢を五分に戻す」ためのものであり、「優勢をさらに広げる」ものにしない。
- トレースバックは解析100%という重い前提、ゼロデイは劣勢でしか溜まらない、キルスイッチは自軍も罰を受ける——いずれも「軽率には撃てない」。
- 結果として、逆転札が決まっても「そこから改めて五分の勝負」になり、撃ったプレイヤーの **その後の作戦ポリシーの質** が最終的な勝敗を決める。運ではなく設計と判断で逆転する。

## 4. AI連携 3レイヤー概要

本作の作戦AIは、プレイヤーの習熟度・目的に応じて3つの実装レイヤーから選べる。**いずれもサーバー権威・決定論的判定** の上で動く（詳細は `ai-interface.md` 参照）。

| レイヤー | 名称 | 実行環境 | 想定ユーザー | 概要 |
| --- | --- | --- | --- | --- |
| **L1** | 組み込みルールエンジン | サーバー内蔵 | 全プレイヤー（入門〜上級） | 第2章のIF-THENを内蔵ビヘイビアツリーとして評価。サーバー権威・決定論的で最も公平。GUIでカード編集 |
| **L2** | 簡易DSL / ノードグラフ | サーバー側サンドボックス | 中〜上級・自作派 | 軽量DSL（またはビジュアルノードグラフ）で、L1より複雑な条件分岐・状態保持ロジックを記述。サンドボックスで安全に実行 |
| **L3** | 自前LLM / 外部AIエンドポイント | プレイヤー側 + サーバー検証 | 上級・研究/開発者（**本企画の主役**） | プレイヤーが用意したLLM（ローカル/外部）を作戦AIとして接続。サーバーが部分観測の盤面状態を送り、LLMが行動を返す。**詳細仕様は `ai-interface.md`** |

### レイヤー間の関係

- L1/L2/L3は **同一ルール・同一サーバー判定** の上で対戦可能（混在マッチも設計上可能）。
- どのレイヤーでも、行動はサーバーが検証・適用する。CP超過・不正行動・見えない敵への攻撃は一律拒否され、勝敗判定はサーバーが権威を持つ。
- L3の自前LLM連携が本作の中核的な遊び・拡張ポイントであり、その重点仕様を次のドキュメントに記す。

→ **続き: [`ai-interface.md`](./ai-interface.md)**
